Cryptographic Algorithms Educational Info

Symmetric Algorithms

Brief History: Adopted in 2001 as a replacement for DES, AES is a widely used symmetric encryption standard.

Pseudocode:

For each block in plaintext:
  Substitute bytes using S-box
  Shift rows
  Mix columns
  Add round key
Repeat for multiple rounds

How it works: AES encrypts data in fixed-size blocks using substitution and permutation steps controlled by a secret key.

Common Use Cases: Secure data transmission, file encryption, VPNs.

Brief History: Developed in the 1970s, DES was an early symmetric-key algorithm but is now considered insecure due to short key length.

Pseudocode:

Initial permutation
For 16 rounds:
  Expand and permute half-block
  XOR with subkey
  Substitute using S-boxes
  Permute
Swap halves
Final permutation

How it works: DES uses a series of permutations and substitutions on 64-bit blocks with a 56-bit key.

Common Use Cases: Legacy systems, educational purposes.

Brief History: Designed in 2008 as a fast and secure stream cipher alternative to AES.

Pseudocode:

Initialize state with key, nonce, counter
For 20 rounds:
  Quarter round operations on state
Add original state to result
Output keystream

How it works: ChaCha20 generates a pseudorandom keystream which is XORed with plaintext for encryption.

Common Use Cases: Secure communications, VPNs, TLS.

Asymmetric Algorithms

Brief History: Developed in 1977, RSA is one of the first public-key cryptosystems widely used for secure data transmission.

Pseudocode:

Generate large primes p, q
Compute n = p * q
Compute totient = (p-1)*(q-1)
Choose e coprime to totient
Compute d as modular inverse of e mod totient
Encrypt: ciphertext = plaintext^e mod n
Decrypt: plaintext = ciphertext^d mod n

How it works: RSA uses a pair of keys (public and private) for encryption and decryption based on modular exponentiation.

Common Use Cases: Secure email, digital signatures, key exchange.

Brief History: ECC emerged in the 1980s as a public-key cryptography approach using elliptic curves, offering similar security with smaller keys.

Pseudocode:

Select elliptic curve and base point G
Generate private key d
Compute public key Q = d * G
Sign/verify or encrypt/decrypt using curve operations

How it works: ECC relies on the difficulty of the elliptic curve discrete logarithm problem for security.

Common Use Cases: Mobile devices, SSL/TLS, cryptocurrencies.

Hashing Algorithms

Brief History: Part of the SHA-2 family, standardized in 2001, widely used for data integrity and security.

Pseudocode:

Initialize hash values
Process message in 512-bit chunks:
  Prepare message schedule
  Compression function with bitwise operations
Update hash values
Output final hash

How it works: Produces a fixed-size 256-bit hash from input data, resistant to collisions.

Common Use Cases: Digital signatures, certificates, blockchain.

Brief History: Developed in 1995, now considered weak due to vulnerabilities but still used in legacy systems.

Pseudocode:

How it works: Produces a 160-bit hash, vulnerable to collision attacks.

Common Use Cases: Legacy applications, some version control systems.

Common Use Cases: High-security applications, digital signatures.
